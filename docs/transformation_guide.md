# m.ppomppu.co.kr HTML 로컬화 변환 가이드

본 문서는 `m.ppomppu.co.kr` 모바일 페이지를 한글 인코딩이 정상적으로 적용된 완전 환 기능의 로컬 버전으로 변환하는 데 사용된 "공식" 및 절차를 설명합니다.

## 1. 문자 인코딩 변환
- **기존**: `euc-kr`
- **변환**: 파일을 `UTF-8` 형식으로 다시 저장하고 메타 태그를 다음과 같이 업데이트합니다.
  ```html
  <meta http-equiv="Content-Type" content="text/html; charset=utf-8" />
  ```

## 2. 리소스 경로 미러링 (변환 공식)
정적 리소스와 동적으로 로드되는 자바스크립트(JS) 리소스가 로컬 환경에서 모두 정상 작동하도록 두 가지 미러링 전략을 사용합니다.

### A. 루트 상대 경로 (내부 경로)
`/js/common.js`와 같이 `/`로 시작하는 절대 경로는 로컬 프로젝트의 루트 디렉토리에 동일한 구조로 미러링합니다.
- **규칙**: `https://m.ppomppu.co.kr/path/to/file` -> `./path/to/file`
- **결과**: HTML 내의 경로를 수정하지 않아도 로컬 서버에서 리소스를 찾을 수 있으며, `G_MOB_URL + '/js/...'`와 같은 동적 JS 호출도 수정 없이 그대로 작동합니다.

### B. 외부 도메인 경로 (CDN 및 GPT)
`cdn2.ppomppu.co.kr`, `gpt.js` 등 외부 도메인에서 제공되는 리소스는 별도의 미러링 디렉토리에 다운로드합니다.
- **규칙**: `//domain.com/path/to/file` -> `/_external/domain_com/path/to/file`
- **결과**: HTML 내의 해당 경로를 로컬 `/_external/...` 경로로 업데이트합니다.

## 3. CSS 자산의 재귀적 로컬화
CSS 파일 내부의 `url()` 참조(폰트, 배경 이미지 등)를 처리합니다.
- **규칙**: 다운로드된 모든 `.css` 파일을 검사하여 `url(...)` 참조를 찾고, CSS 파일 위치를 기준으로 경로를 해석하여 미러링 구조에 맞게 다운로드합니다.

## 4. 자바스크립트 전역 변수 재정의
뽐뿌는 전역 변수를 사용하여 동적 자산 URL을 생성합니다. 이를 로컬 루트를 가리키도록 `<head>` 영역에서 재정의합니다.
- `G_IMG_URL`, `G_CDN_URL`, `G_STATIC_URL`, `G_WWW_URL` -> `""` (빈 문자열)
- **효과**: `G_IMG_URL + "/images/foo.png"`가 `/images/foo.png`(로컬 경로)로 해석되어 정상적으로 로드됩니다.

## 5. 렌더링 및 확장자 정규화
- **무명 iframe 리소스**: 광고 배너와 같이 확장자가 없는 리소스(예: `//adex.ednplus.com/...`)는 브라우저에서 올바르게 렌더링되도록 로컬 저장 시 `.html` 확장자를 부여합니다.
- **경로 클리닝**: 미러링 과정에서 생성된 윈도우 스타일의 백슬래시(`\`)를 모두 표준 슬래시(`/`)로 변환하여 브라우저 호환성을 보장합니다.

## 요약
1. **인코딩**: UTF-8로 변경.
2. **미러링**: 서버 디렉토리 구조를 로컬 루트에 복제.
3. **캡처**: CSS 내부의 하위 리소스까지 추적 및 다운로드.
4. **상대화**: 전역 JS 변수를 로컬 루트로 강제 매핑.
5. **정규화**: 경로 문자열 정제(백슬래시 제거, 이중 슬래시 교정).
