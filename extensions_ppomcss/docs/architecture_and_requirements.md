# PpomCSS Audit 확장 프로그램 - 사양 및 아키텍처 가이드

이 문서는 2026년 3월 현재 PpomCSS Audit 크롬 확장 프로그램의 요구사항, 설계 및 구현 세부 사항을 다룹니다. 추후 리팩토링 및 유지보수 시 참고용으로 작성되었습니다.

## 1. 핵심 목표
- **CSS 감사 (Audit)**: 뽐뿌 모바일 사이트의 특정 페이지에서 실제 사용 중인 CSS 규칙과 사용되지 않는 규칙을 식별합니다.
- **다중 페이지 병합 (Merge)**: 여러 페이지(게시판 목록, 읽기, 메인 등)에서 수집된 감사 데이터를 하나로 합쳐, 중복이 제거되고 최적화된 단일 CSS 파일을 생성합니다.
- **자동화 (Automation)**: "Auto Scan" 기능과 전역 설정을 통해 반복적인 감사 작업을 효율화합니다.

## 2. 주요 기능

### 2.1 감사 및 선택 (Audit & Selection)
- **단일 선택 워크플로우**: 실수를 방지하고 집중도를 높이기 위해 한 번에 하나의 스타일시트(예: `mobile.css`)만 선택하여 최적화 작업을 진행합니다.
- **전역 선택 유지 (Global Persistence)**: 한 페이지에서 선택한 파일은 다른 게시판이나 페이지로 이동해도 그대로 유지됩니다.
- **가독성 높은 출력 (Pretty Format)**: 추출된 모든 CSS는 4스페이스 들여쓰기와 속성별 줄바꿈이 적용된 전문가용 포맷으로 제공됩니다.

### 2.2 자동화 (Auto Scan)
- **내비게이션 트리거**: 활성화 시, 페이지 이동이나 탭 업데이트 후 1초 뒤에 자동으로 CSS 감사를 실행합니다.
- **자동 저장**: 스캔 성공 시 결과가 'Merge History'에 자동으로 추가됩니다.

### 2.3 병합 히스토리 및 최적화 (Merge History)
- **파일명별 그룹화**: 저장된 스캔 결과는 원본 파일명별로 섹션이 나뉘어 표시됩니다 (예: `MOBILE.CSS (4 pages)`).
- **게시판 ID별 분류 요약 (Classify Summary)**: 'Classify' 버튼을 누르면 별도의 팝업(Modal)이 뜨면서, 각 게시판 ID(`id=freeboard` 등)별로 스캔된 페이지 수, 추출된 규칙 수, 합산 용량 등의 통계 정보를 요약하여 제공합니다.
- **중복 제거 (Deduplication)**: `Set` 자료구조를 활용하여 여러 페이지에서 공통적으로 사용되는 규칙이 최종 결과물에 단 한 번만 포함되도록 보장합니다.
- **메타데이터 제공**: 병합 결과물 상단에 최종 용량, 병합된 페이지 수, 중복 제거된 총 규칙(rules) 수를 표시합니다.

### 2.4 사용자 피드백 및 인코딩
- **실시간 누적 카운트**: 스캔이 완료될 때마다 현재 선택된 파일에 대해 총 몇 개의 데이터가 히스토리에 쌓였는지 버튼에 바로 표시됩니다 (예: `Saved (Total 5 pages)`).
- **한글 인코딩 지원**: 뽐뿌 사이트의 특성에 맞춰 `fetch` 시 `EUC-KR` 인코딩을 자동으로 감지하고 처리하여, CSS 내의 한글 주석이나 폰트 설정이 깨지지 않도록 보장합니다.

## 3. 기술적 아키텍처

### 3.1 구성 요소
- **사이드 패널 (`sidepanel.js/html`)**: 확장 프로그램의 컨트롤 타워입니다. UI 상태 관리, `localStorage` 제어, 스타일시트 직접 호출(CORS 우회), `TextDecoder`를 통한 인코딩 처리, CSS 병합 및 정렬 로직을 담당합니다.
- **컨텐츠 스크립트 (`content_script.js`)**: 페이지 컨텍스트에서 실행됩니다. `document.querySelector`를 사용하여 실제 DOM에 스타일 규칙이 적용되었는지 검사합니다.
- **백그라운드 스크립트 (`background.js`)**: 사이드 패널의 활성화 상태와 Auto Scan 트리거(`chrome.tabs.onUpdated`)를 관리합니다.

### 3.2 데이터 저장 방식 (`localStorage`)
- `auditHistory`: URL, 타임스탬프, 그룹화된 규칙들이 포함된 객체 배열입니다.
- `globalSelectedSheet`: 현재 전역적으로 선택된 타겟 스타일시트의 파일명입니다.
- `autoScanEnabled`: 내비게이션 기반 자동 스캔 활성화 여부입니다.

## 4. 최적화 워크플로우
1. **대상 설정**: 최적화할 스타일시트(예: `mobile.css`)를 선택합니다.
2. **데이터 수집**: `Auto Scan`을 켜고 필요한 게시판의 여러 페이지(목록, 뷰, 본문 등)를 방문합니다.
3. **요약 확인**: 'Merge History'에서 'Classify' 버튼을 눌러 각 게시판별로 데이터가 충분히 모였는지 확인합니다.
4. **병합 및 배포**: `Merge & Deduplicate`를 눌러 최적화된 코드를 복사하고 운영 환경에 반영합니다.

## 5. 향후 리팩토링 참고 사항
- **모듈화**: 히스토리 관리 및 CSS Prettifier 로직을 별도의 유틸리티 모듈로 분리할 필요가 있습니다.
- **성능 최적화**: DOM이 초대형인 경우 `content_script.js`의 선택자 검사 루프를 최적화해야 합니다.
- **동기화**: `localStorage` 용량 제한에 대비하여 `chrome.storage.local`로의 전환을 고려할 수 있습니다.
