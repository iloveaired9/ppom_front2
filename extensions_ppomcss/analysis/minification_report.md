# CSS 최적화 및 경량화 보고서 (Minification Report)

본 문서는 `main_mobile.css`를 기반으로 실제 `m_ppom_dark.html`에서 사용되는 요소들만 추출하여 최적화 및 경량화한 과정과 결과를 기술합니다.

## 1. 최적화 결과 요약

## 2. 최적화 방법론 (Methodology)

### 1) 사용 요소 추출
- `m_ppom_dark.html` 파일을 분석하여 실제 사용 중인 **모든 Class(124개) 및 ID(64개)** 리스트를 확보했습니다.
- HTML 기본 Tag(body, div, ul, li 등)에 대한 스타일은 레이아웃 유지를 위해 모두 보존했습니다.

### 2) CSS 필터링 (Tree Shaking)
- Python 스크립트를 사용하여 다음 규칙에 해당하는 CSS Block만 추출했습니다.
  - 선택자(Selector)에 추출된 Class 또는 ID가 포함된 경우
  - 선택자가 HTML 기본 Tag인 경우
  - `:root` 변수 및 CSS 전역 변수(`.--`) 정의 부문

### 3) 코드 정리 및 인코딩 정규화 (Cleanup & Encoding)
- **주석 제거**: 모든 CSS 주석(`/* ... */`)을 제거하여 인코딩 충돌 가능성을 원천 차단했습니다.
- **UTF-8 변환**: 원본의 `CP949` 인코딩을 `UTF-8`로 변환하여 생성함으로써 현대적인 웹 환경(HTML5)에서의 호환성을 확보했습니다.
- **최종 최적화**: 불필요한 공백과 개행을 제거하여 전송 효율을 극대화했습니다.

## 3. 주요 보존 및 제거 내역

### 주요 보존 (Preserved)
- **전역 설정**: `:root` 변수(다크모드 컬러 포함), Reset 스타일, 폰트 정의.
- **핵심 UI**: 헤더(`.hd`), 메인 리스트(`.list_main`, `.bbsList`), 푸터(`.site-footer`), 검색바, 탭 메뉴.
- **다크모드 변수**: 뽐뿌 순정 다크모드 구현에 필요한 모든 `--color-m-dark-*` 변수.

### 주요 제거 (Pruned)
- **특정 게시판 전용 스타일**: 중고장터, 뽐뿌게시판 외의 특수 게시판 코드.
- **이벤트/특수 기능**: 실명인증, 회원가입(`.join_box`), 쿠폰(`.coupon-history`), 랭킹 페이지 등 메인 페이지에서 노출되지 않는 방대한 스타일.
- **레거시 코드**: 사용되지 않는 구버전 브라우저 대응용 중복 스타일.

## 4. 활용 제언
`main_mobile_min.css`는 원본 대비 3분의 1 수준의 크기로, **초기 로딩 속도(LCP)**를 획기적으로 개선할 수 있습니다. 
로컬라이징된 다크모드 페이지(`m_ppom_dark.html`, `m_naver_dark.html`)에서 실제 상용 서빙 시 이 파일을 사용하면 사용자 경험(UX) 측면에서 훨씬 유리합니다.
